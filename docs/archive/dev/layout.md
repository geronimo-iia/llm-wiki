# Repository Layout — Developer Guide

See `docs/design/repository-layout.md` for the design rationale. This document
covers the implementation details.

---

## Slug resolution

A slug is always a path without extension relative to the wiki root. Two disk
forms exist:

| Form | Disk path | When used |
|------|-----------|-----------|
| Flat | `{slug}.md` | Page with no assets |
| Bundle | `{slug}/index.md` | Page with co-located assets |

Resolution order (checked by `resolve_slug`):

1. `{wiki_root}/{slug}.md` — flat form wins if it exists
2. `{wiki_root}/{slug}/index.md` — bundle form

```rust
pub fn resolve_slug(wiki_root: &Path, slug: &str) -> Option<PathBuf>
```

Returns `None` if neither form exists. All walkers (`search`, `graph`,
`context`, `server`) call `resolve_slug` instead of hard-coding `.md`.

---

## slug_for

Derives the slug from a file path:

```rust
pub fn slug_for(path: &Path, wiki_root: &Path) -> String
```

- `{wiki_root}/concepts/foo/index.md` → `"concepts/foo"`
- `{wiki_root}/concepts/foo.md` → `"concepts/foo"`
- `{wiki_root}/concepts/foo/diagram.png` → not a page; callers skip non-`.md` files

All walkers that previously used `rel.with_extension("").to_string_lossy()` now
call `slug_for` so bundle pages get the correct slug.

## Bundle promotion

`promote_to_bundle(wiki_root, slug)` converts a flat page to a bundle:

```
Before:  {wiki_root}/{slug}.md
After:   {wiki_root}/{slug}/index.md
```

Steps:
1. If `{slug}/index.md` already exists → no-op, return `Ok(())`.
2. If `{slug}.md` does not exist → return an error.
3. `create_dir_all({slug}/)` then `rename({slug}.md → {slug}/index.md)`.

The slug is unchanged after promotion. `resolve_slug` finds the bundle form
automatically.

`write_asset_colocated` calls `promote_to_bundle` automatically before writing
the asset, so callers never need to promote manually.

## Asset placement decision

| Situation | Placement | Function |
|-----------|-----------|----------|
| Asset belongs to one page | `{page_slug}/{filename}` | `write_asset_colocated` |
| Asset shared by 2+ pages | `assets/{subdir}/{filename}` | `write_asset_shared` |

Kind → subdir mapping for shared assets:

| Kind | Extensions | Subdir |
|------|------------|--------|
| `image` | `.png`, `.jpg`, `.svg`, `.gif` | `assets/diagrams/` |
| `yaml` | `.yaml`, `.yml` | `assets/configs/` |
| `toml` | `.toml` | `assets/configs/` |
| `json` | `.json` | `assets/configs/` |
| `script` | `.py`, `.sh`, `.rs`, `.js` | `assets/scripts/` |
| `data` | `.csv`, `.tsv`, `.jsonl` | `assets/data/` |
| `other` | anything else | `assets/other/` |

## assets/index.md

`assets/index.md` is a committed Markdown table of all shared assets. It is
regenerated by `regenerate_assets_index(wiki_root)` after every call to
`write_asset_shared`.

Format:

```markdown
# Shared Assets

| slug | kind | caption | referenced_by |
|------|------|---------|---------------|
| assets/diagrams/chart | image | | |
| assets/configs/cfg | yaml | | |
```

Bundle assets are **not** in this index — they are discoverable via the bundle
folder and referenced by short relative paths in the page body.

## Walker rules

All walkers that enumerate pages must apply these filters:

1. Only process `.md` files.
2. Skip `.wiki/` and `raw/`.
3. For a `.md` file that is **not** named `index.md`: if its parent directory
   contains an `index.md`, skip it — it is a co-located asset, not a page.
4. Use `slug_for(path, wiki_root)` to derive the slug.

```rust
// Skip non-index.md files inside bundle folders.
if let Some(filename) = path.file_name() {
    if filename != "index.md" {
        if let Some(parent) = path.parent() {
            if parent.join("index.md").exists() {
                continue;
            }
        }
    }
}
let slug = slug_for(path, wiki_root);
```

## Orphan asset ref lint

`lint::lint` scans every `index.md` in the wiki for `](./filename)` references
and checks whether `filename` exists beside `index.md`. Missing files are
reported in `LintReport::orphan_asset_refs` and in the `## Orphan Asset
References` section of `LINT.md`.
